my-messenger/프로젝트 루트
프로젝트 최상위 폴더. 모든 명령어는 여기서 실행해요. server/는 백엔드, src/는 프론트엔드 코드가 들어가요.
server/기혁
백엔드 전체 폴더. 기혁이 관리해요. Node.js + Express 서버 코드가 들어가요.
기혁 전담
app.js기혁지후 협업
Express 서버 진입점. 기혁이 미들웨어·라우터를 등록하고, 지후가 Socket.IO 초기화 코드를 이 파일에 추가해요.

기혁 담당: app.use(express.json()), app.use('/api', router)
지후 담당: const io = require('socket.io')(httpServer)
기혁지후
routes/기혁
API 엔드포인트(주소) 파일들이 들어가요. 프론트가 요청하는 모든 주소를 여기서 만들어요.
기혁 전담
auth.js기혁
POST /api/auth/register (회원가입), POST /api/auth/login (로그인·JWT 발급). 4번 영민이 Axios로 이 주소를 호출해요.
messages.js기혁
GET /api/messages/:roomId (메시지 목록), POST /api/messages (메시지 저장). 영민의 React Query 훅이 이 주소를 사용해요.
users.js기혁서현 협업
GET /api/users/search?q=검색어 (사용자 검색), POST /api/friends (친구 추가). 기혁이 만들고, 서현이 프론트에서 호출해요.
기혁 제작서현 호출
models/기혁
MongoDB 데이터 구조(Schema)를 정의하는 폴더. DB에 어떤 형태로 저장할지 설계해요. 처음에 잘 설계해야 나중에 안 바꿔도 돼요.
기혁 전담
User.js기혁
사용자 스키마: name, email, password(암호화), friends(친구 목록), profileImage 필드를 정의해요.
Message.js기혁
메시지 스키마: content(내용), sender(보낸 사람 ID), room(채팅방 ID), isRead(읽음 여부), createdAt 필드.
Room.js기혁
채팅방 스키마: participants(참여자 배열), lastMessage(마지막 메시지), createdAt 필드.
socket/지후
Socket.IO 서버 이벤트 핸들러 폴더. 지후가 전담해요.
지후 전담
index.js지후
소켓 이벤트 핸들러 모음: joinRoom, sendMessage, disconnect, readMessage 이벤트를 여기서 처리해요.
middleware/기혁
API 요청마다 실행되는 공통 처리 코드. JWT 토큰 검증 미들웨어가 들어가요. 이게 없으면 로그인 없이도 API를 쓸 수 있어서 위험해요.
기혁 전담
.env기혁절대 Git 금지
서버 비밀 설정. PORT=5000, MONGODB_URI=mongodb+srv://..., JWT_SECRET=비밀키. .gitignore에 반드시 추가해야 해요!
src/프론트 공통
프론트엔드 React 코드 폴더. 예지·영민·서현·지후가 함께 작업하는 공간이에요.
api/영민
Axios API 요청 함수들이 들어가요. 영민이 전담해요. 다른 팀원은 여기 함수를 가져다 써요.
영민 전담
client.js영민
Axios 인스턴스 생성 + 인터셉터 설정. 모든 요청에 JWT 토큰을 자동으로 붙이고, 401 에러 시 자동 로그아웃 처리하는 핵심 파일이에요.
authApi.js영민
login(email, password), register(name, email, password) 함수. 로그인 화면(예지)에서 이 함수를 불러와 사용해요.
messageApi.js영민
getMessages(roomId), sendMessage(content, roomId) 함수. React Query 훅에서 사용해요.
userApi.js영민서현 사용
searchUsers(query), addFriend(userId) 함수. 영민이 만들고, 서현이 SearchUser 컴포넌트에서 가져다 써요.
영민 제작서현 사용
store/영민
Redux 상태관리 폴더. 앱 전체에서 공유하는 데이터(로그인 정보, 메시지 목록)를 관리해요.
영민 전담
index.js영민
Redux store 설정. authSlice, messageSlice를 합쳐서 하나의 store로 만드는 파일이에요.
authSlice.js영민
로그인 상태(user, token, isLoggedIn)를 Redux로 관리하는 파일. 로그인·로그아웃 액션을 처리해요.
messageSlice.js영민
메시지 목록 상태를 관리. 소켓으로 새 메시지가 오면 이 slice에 추가해서 화면에 실시간 반영해요.
hooks/영민지후 협업
React 커스텀 훅 폴더. 영민이 API 훅을, 지후가 소켓 훅을 만들어요.
영민지후
useMessages.js영민
React Query로 메시지 목록을 불러오는 훅. 로딩·에러 상태를 자동 관리하고, 3초마다 자동으로 새 메시지를 불러와요.
useSocket.js지후
Socket.IO 연결·이벤트 리스닝 훅. 컴포넌트가 마운트되면 소켓 연결, 언마운트되면 연결 해제하는 로직이 들어가요.
pages/예지서현 일부
각 화면(페이지) 컴포넌트 폴더. 예지가 주요 페이지 틀을 잡고, 서현이 친구·프로필 페이지를 담당해요.
예지서현
LoginPage.jsx예지
로그인·회원가입 화면 레이아웃. 예지가 UI를, 영민이 실제 API 연결(authApi.login() 호출)을 담당해요.
ChatPage.jsx예지
메인 채팅 화면. Sidebar + ChatWindow를 배치하는 레이아웃 파일. Tailwind의 flex로 좌우 배치해요.
FriendsPage.jsx서현
친구 목록 전체 페이지. FriendList, SearchUser 컴포넌트를 조합해서 화면을 구성해요. 서현이 담당해요.
ProfilePage.jsx서현
내 프로필 보기·수정 페이지. 이름·상태메시지 수정, 프로필 사진 업로드 기능이 들어가요.
components/예지서현 일부
재사용 가능한 UI 조각 폴더. 예지가 채팅 관련 컴포넌트를, 서현이 친구·부가기능 컴포넌트를 만들어요.
예지서현
MessageBubble.jsx예지
메시지 말풍선. isMine props로 내 메시지(오른쪽·보라색)와 상대 메시지(왼쪽·회색)를 다르게 표시해요.
MessageInput.jsx예지
하단 메시지 입력창 + 전송 버튼. onSend props로 함수를 받아서 실행해요. 실제 전송 로직은 영민이 연결해줘요.
Sidebar.jsx예지
좌측 사이드바 틀(레이아웃). 안에 들어갈 RoomList는 서현이 만들어요. 예지는 사이드바의 위치·크기·스타일을 담당해요.
RoomList.jsx서현
채팅방 목록 컴포넌트. 마지막 메시지 미리보기, 읽지 않은 메시지 수 배지를 표시해요. 예지의 Sidebar 안에 들어가요.
SearchUser.jsx서현
사용자 검색 입력창 + 검색 결과 목록. 입력할 때마다 300ms 기다렸다가 API 요청해요(debounce). 영민의 userApi.searchUsers()를 사용해요.
Notification.jsx서현
알림 팝업·배지. 지후의 소켓 이벤트(newMessage)를 받아서 알림을 트리거해요. 브라우저 Notification API도 활용할 수 있어요.
App.jsx영민진입점
React 앱 최상위. 영민이 여기서 QueryClientProvider와 Redux Provider를 감싸줘야 해요. 이걸 안 하면 React Query·Redux가 아예 작동 안 해요.
main.jsx건드릴 일 없음
React 앱을 HTML에 붙이는 코드. 거의 건드릴 일 없어요.
tailwind.config.js예지
Tailwind CSS 설정 파일. 예지가 프로젝트 색상·폰트를 커스텀해요. content 경로를 꼭 설정해야 스타일이 적용돼요.
package.json자동 관리
설치된 라이브러리 목록. npm install 패키지명 하면 자동으로 기록돼요. 직접 수정할 일은 거의 없어요.
.env영민절대 Git 금지
프론트 비밀 설정. VITE_API_URL=http://서버주소/api. Vite에서 환경변수는 반드시 VITE_로 시작해야 해요. Git에 올리면 안 돼요!
vite.config.js영민
Vite 개발 서버 설정. CORS 에러 날 때 여기에 proxy 설정을 추가해요. 기혁 서버 주소를 여기에 연결해서 테스트해요.
JavaScript .js

웹의 기본 프로그래밍 언어. 버튼 클릭, 데이터 처리 등 모든 동작을 담당해요.

const name = "김민준";
const greet = () => `안녕 ${name}`;
기혁지후영민
Node.js 서버 실행 환경

JavaScript를 서버에서 실행하는 환경. 브라우저 없이도 JS 코드를 돌릴 수 있어요. Express와 함께 써요.

// 서버에서 돌아가는 JS
app.get('/api/hello', (req, res) => {
  res.json({ msg: "안녕!" });
});
기혁 주 사용
JSX .jsx

JavaScript + HTML을 합친 것. React에서 화면을 만들 때 써요. JS 안에 HTML을 그냥 쓸 수 있어요.

return (
  <div>안녕하세요</div>
);
예지영민서현
Tailwind CSS className

CSS를 클래스 이름으로 바로 적용하는 방식. 별도 CSS 파일 없이 className에 스타일을 써요.

<div className="flex gap-4 p-6
 bg-purple-600 rounded-xl">
예지 주 사용서현 사용
JSON .json

데이터를 저장하고 주고받는 형식. 서버에서 메시지를 받을 때 이 형태로 와요.

{
  "sender": "기혁",
  "content": "안녕!"
}
전원 관련
Socket.IO 실시간 통신

서버-클라이언트 간 실시간 연결. 새로고침 없이 메시지가 즉시 오가게 해요. 카카오톡이 이 방식을 써요.

socket.emit('sendMessage', data);
socket.on('receiveMessage',
  (msg) => setMsgs([...msgs, msg]));
지후 주 사용
async/await

서버에 데이터 요청할 때 쓰는 문법. 답이 올 때까지 기다렸다가 다음 코드를 실행해요.

const getData = async () => {
  const res = await api.get(url);
  // res가 올 때까지 대기
};
기혁영민서현
useState / Hook

React에서 변하는 데이터(상태)를 관리하는 기능. 메시지 목록, 입력값 등을 저장해요.

const [msgs, setMsgs] =
  useState([]);
// msgs: 현재값, setMsgs: 변경
예지영민서현
Mongoose Schema DB 설계

MongoDB에 저장할 데이터의 구조를 정의하는 문법. 어떤 필드가 있는지, 타입은 무엇인지 정해요.

const userSchema = new Schema({
  name: String,
  email: { type: String, unique: true }
});
기혁 주 사용
Redux Toolkit 상태관리

앱 전체에서 공유하는 데이터를 한 곳에서 관리. 로그인 정보를 어느 화면에서도 쓸 수 있게 해줘요.

const authSlice = createSlice({
  name: 'auth',
  initialState: { user: null },
  reducers: { setUser: ... }
});
영민 주 사용
처음 한 번만 실행 최초 1회
mkdir server && cd server && npm init -y 기혁 백엔드 초기화
기혁 전용. 백엔드 폴더를 만들고 Node.js 프로젝트를 초기화해요. package.json이 자동 생성돼요. 프로젝트 루트에서 실행해요.
npm install express mongoose bcryptjs jsonwebtoken cors dotenv 기혁 백엔드 패키지
기혁 전용 — server/ 폴더에서 실행.
express: 서버 프레임워크 / mongoose: MongoDB 연결 / bcryptjs: 비밀번호 암호화 / jsonwebtoken: JWT 발급 / cors: 프론트-백엔드 연결 허용 / dotenv: .env 파일 읽기
npm install --save-dev nodemon 기혁 자동 재시작 도구
기혁 전용 — server/ 폴더에서 실행. 코드를 수정하면 서버가 자동으로 재시작돼요. --save-dev는 개발할 때만 쓰는 도구라는 표시예요.
npm install socket.io 지후 서버 소켓
지후 전용 — server/ 폴더에서 실행. 서버 측 Socket.IO 설치. 기혁이 서버 세팅을 마친 뒤 지후가 server/ 폴더에서 실행해요.
npm create vite@latest . 예지 프론트 프로젝트 생성
예지 또는 기혁이 한 번만 실행. React 프론트엔드 프로젝트 생성. .은 현재 폴더에 만들어라는 뜻. React + JavaScript 선택!
npm install -D tailwindcss postcss autoprefixer && npx tailwindcss init -p 예지 Tailwind 설치
예지 전용 — 프론트 루트에서 실행. Tailwind CSS 설치 후 설정 파일 생성. 실행하면 tailwind.config.js와 postcss.config.js가 생겨요.
npm install axios @tanstack/react-query react-redux @reduxjs/toolkit socket.io-client 영민지후 프론트 패키지
프론트 루트에서 한 번만 실행. 영민·지후가 쓰는 핵심 패키지를 한꺼번에 설치해요.

axios: API 요청 (영민) / @tanstack/react-query: 서버 데이터 관리 (영민) / react-redux @reduxjs/toolkit: 상태관리 (영민) / socket.io-client: 소켓 연결 (지후)
npm install 전원 처음 받을 때
전원 공통 — Git clone 후 첫 번째로 실행. package.json에 기록된 라이브러리를 모두 설치해요. node_modules/ 폴더가 생기면 성공!
매일 개발할 때 자주 사용
nodemon app.js 기혁지후 백엔드 서버 시작
server/ 폴더에서 실행. 코드를 수정하면 서버가 자동으로 재시작돼요. 기혁이 서버를 켜야 지후·영민·서현이 API 테스트를 할 수 있어요. http://localhost:5000에서 실행돼요.
npm run dev 프론트 전원 개발 서버 시작
프론트 루트에서 실행. 예지·영민·서현·지후 모두 화면 작업할 때 사용. 코드 수정 시 브라우저가 자동 새로고침돼요. http://localhost:5173에서 확인해요.
Ctrl + C 전원 서버 종료
터미널에서 실행 중인 서버를 끄는 단축키. 명령어가 아니라 단축키예요! 터미널 창 클릭 후 Ctrl+C를 눌러요.
Git 관련 기혁 주관전원 숙지
git init 기혁 Git 저장소 초기화
기혁이 최초 1회만 실행. 프로젝트 폴더에 Git 저장소를 만들어요. GitHub 레포와 연결하기 전에 실행해요.
git checkout -b feature/기능이름 전원 새 브랜치 생성
작업 전 새 브랜치를 만들어요. 예: git checkout -b feature/login (기혁), git checkout -b feature/socket (지후). main 브랜치에서 직접 작업하면 충돌이 생겨요!
git add . && git commit -m "feat: 설명" 전원 변경사항 저장
변경된 파일을 스테이징하고 커밋해요. 커밋 메시지 규칙: feat: 새 기능, fix: 버그 수정, style: UI 스타일. 예: feat: 로그인 API 연결
git push origin feature/기능이름 전원 GitHub에 올리기
내 브랜치 코드를 GitHub에 올려요. 올린 뒤 GitHub에서 Pull Request(PR)를 만들면 기혁이 검토 후 merge해줘요.
git pull origin main 전원 최신 코드 받기

기혁 — 백엔드 · 총괄
팀 전체의 뼈대를 만드는 역할. 서버·DB·API를 구축하고 Git으로 팀 코드를 통합 관리해요.
주요 작업 상세
서버 구축
Node.js + Express로 서버 파일 만들기. app.js에서 포트 설정, 라우터 연결, 미들웨어 등록. 서버가 없으면 다른 팀원이 아무것도 못 해요.
DB 설계
MongoDB에 어떤 데이터를 저장할지 구조(Schema)를 설계. User, Message, Room 3개 컬렉션이 핵심. 나중에 바꾸면 전체 코드가 바뀌니 처음이 중요해요.
로그인·회원가입
비밀번호를 bcrypt로 암호화하고, 로그인 성공 시 JWT 토큰 발급. 이 토큰을 4번 담당자가 받아서 로그인 유지에 씀.
API 제작
프론트가 요청할 수 있는 주소(엔드포인트)를 만들어요. 4번·5번 담당자가 Axios로 호출할 주소를 다 여기서 만들어요.
Git 관리 (총괄)
GitHub 레포지토리 생성, 브랜치 전략 설정(main / dev / feature/xxx). 팀원 코드를 Pull Request로 받아서 충돌 해결 후 merge. 팀 전체의 코드 흐름을 관리해요.
작업 순서
GitHub 레포 생성·브랜치 설정
Express 서버 기본 세팅
MongoDB 연결 + Schema 설계
회원가입·로그인 API
메시지·채팅방 API
팀원 PR 리뷰·merge
핵심 코드 예시
// POST /api/auth/login
router.post('/login', async (req, res) => {
  const user = await User.findOne({ email: req.body.email });
  const ok = await bcrypt.compare(req.body.password, user.password);
  if (!ok) return res.status(401).json({ msg: '비밀번호 오류' });
  const token = jwt.sign({ id: user._id }, process.env.JWT_SECRET);
  res.json({ token }); // 4번이 이 토큰을 받아서 저장
});
다른 팀원과 협업 포인트
2번(실시간 채팅)에게: Socket.IO 서버 초기화 코드(io = require('socket.io')(server))를 app.js에서 같이 설정해줘야 해요
4번(기능 연동)에게: API 주소 목록(엔드포인트 명세)을 문서로 공유해야 4번이 Axios로 연결할 수 있어요
5번(부가기능)에게: 친구 추가·검색 API도 여기서 만들어줘야 해요
꼭 알아야 할 것
API 명세서를 꼭 만들어요. 노션이나 README에 "POST /api/messages → body: {content, roomId} → response: {id, content, sender, createdAt}" 형식으로 정리해두면 팀 전체가 헷갈리지 않아요.
2
지후 — 실시간 채팅 담당
Socket.IO로 서버-클라이언트 간 실시간 연결을 만드는 역할. 카카오톡처럼 새로고침 없이 메시지가 바로 오가게 해요.
주요 작업 상세
Socket.IO 연결
프론트(socket.io-client)와 백엔드(socket.io)를 연결하는 코드 작성. 사용자가 앱을 열면 자동으로 소켓이 연결돼요.
메시지 송수신
클라이언트가 emit('sendMessage')으로 보내면 서버가 받아서 같은 방 전체에 emit('receiveMessage')로 뿌려줘요.
채팅방 입장
Socket.IO의 room 기능을 사용. 특정 채팅방 ID로 join하면 그 방 사람들끼리만 메시지가 가요.
읽음·접속 상태
사용자가 접속하면 online 상태로 표시, 나가면 offline으로 변경. 메시지 읽으면 읽음 처리 이벤트 발송.
Socket.IO 이벤트 흐름
사용자 접속
connect 이벤트
joinRoom emit
sendMessage emit
서버 → 방 전체 broadcast
receiveMessage on
핵심 코드 예시
// 서버 (백엔드와 협력)
io.on('connection', (socket) => {
  socket.on('joinRoom', (roomId) => socket.join(roomId));
  socket.on('sendMessage', (data) => {
    io.to(data.roomId).emit('receiveMessage', data);
  });
  socket.on('disconnect', () => { /* offline 처리 */ });
});

// 프론트 (3·4번 담당자에게 전달)
const socket = io('http://서버주소');
socket.emit('sendMessage', { roomId, content });
다른 팀원과 협업 포인트
1번(백엔드)에게: app.js에서 HTTP 서버 + Socket.IO 같이 초기화해야 해요. 서버 코드에 소켓 초기화 코드를 추가해달라고 요청해요
3번(UI)에게: 메시지가 오면 화면에 추가되는 UI 컴포넌트를 맞춰야 해요. 어떤 데이터 형태로 넘겨줄지 미리 협의해요
4번(기능 연동)에게: 소켓 연결 훅(useSocket.js)을 같이 만들어요
HTTP vs WebSocket 차이를 기억해요. 일반 API(Axios)는 요청할 때만 연결되지만, Socket.IO는 계속 연결된 통로예요. 메시지 "보내기"는 소켓, 과거 메시지 "불러오기"는 REST API(4번 담당)를 써요.

예지 — 프론트엔드 UI 담당
사용자가 직접 보고 클릭하는 화면을 만드는 역할. 예쁜 디자인보다 쓰기 편한 UX가 더 중요해요.
주요 작업 상세
전체 레이아웃
사이드바(채팅방 목록) + 메인(채팅창) 2단 구조. Tailwind로 flex 배치. 전체 틀을 잡아줘야 다른 팀원이 조각을 끼울 수 있어요.
채팅창 UI
메시지 말풍선 컴포넌트(MessageBubble.jsx), 내 메시지는 오른쪽, 상대는 왼쪽. 스크롤 자동 아래로 내리는 기능도 포함.
로그인·회원가입 화면
입력 폼 UI 제작. 실제 로그인 API 연결은 4번이 해요. 3번은 화면 구조와 스타일만 잡아줘요.
반응형 디자인
모바일에서도 잘 보이게 Tailwind breakpoint 활용(md:flex, hidden md:block 등). 모바일에서 사이드바 숨기고 채팅창만 보이게.

다른 팀원과 협업 포인트
4번(기능 연동)에게: 컴포넌트에 props 형식을 맞춰줘야 해요. 예를 들어 MessageBubble이 받는 message 객체 형태를 서로 맞춰요
2번(실시간 채팅)에게: 새 메시지가 오면 채팅창에 추가하는 UI 업데이트를 어떻게 할지 협의해요
5번(부가기능)에게: 친구 목록·프로필 UI 컴포넌트도 만들어줘야 해요
UI와 로직을 분리해요. 3번은 화면 모양만, 실제 데이터 연결은 4번이 해요. 예를 들어 버튼 클릭 시 어떤 함수를 실행할지는 props로 받는 구조(onSend, onLogin)로 만들면 협업이 쉬워요.
4
영민 — 프론트 기능 연동 담당
3번이 만든 UI에 실제 데이터를 연결하는 역할. 서버와 화면 사이의 다리예요. 이 담당이 없으면 버튼을 눌러도 아무 일도 안 일어나요.
주요 작업 상세
API 연결 (Axios)
src/api/ 폴더에 서버 요청 함수 작성. 1번이 만든 API 주소로 Axios 요청을 보내고 응답 데이터를 화면에 전달해요.
상태 관리 (Redux)
로그인한 유저 정보, 메시지 목록 등 앱 전체에서 쓰는 데이터를 Redux store에서 관리. 어느 컴포넌트에서도 같은 데이터를 볼 수 있어요.
로그인 유지
1번이 준 JWT 토큰을 localStorage에 저장. 새로고침해도 토큰이 있으면 자동 로그인. Axios 요청마다 헤더에 토큰을 붙여줘요.
데이터 처리 (React Query)
과거 메시지 불러오기, 로딩/에러 상태 관리, 자동 재요청 등을 React Query로 처리. 서버 데이터와 화면을 자동으로 동기화해요.
작업 순서
Axios 기본 설정
Redux store 구성
로그인·토큰 처리
React Query 훅 작성
3번 UI에 연결

// Axios 인터셉터 - 모든 요청에 토큰 자동 첨부
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('token');
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// React Query - 메시지 불러오기
export const useMessages = (roomId) => useQuery({
  queryKey: ['messages', roomId],
  queryFn: () => api.get(`/messages/${roomId}`),
});

서현 — 친구·부가기능 담당
기본 채팅 외 사용자 경험을 풍부하게 만드는 기능들. 1·4번이 만든 API를 활용해서 React로 화면을 구현해요.
주요 작업 상세
친구 추가
검색된 사용자에게 친구 요청 전송, 수락/거절 UI 구현. DB에 친구 관계 저장은 1번이 API로 만들어주면, 5번은 버튼 누를 때 그 API 호출.

이름·이메일로 사용자 검색. 입력할 때마다 API 요청하면 서버 부하가 생기니 300ms 딜레이(debounce)를 써요.
채팅방 목록
내가 참여한 채팅방 리스트, 마지막 메시지 미리보기, 읽지 않은 메시지 수 배지 표시. 3번이 만든 Sidebar 컴포넌트에 실제 데이터를 연결해요.
알림 기능
새 메시지·친구 요청 알림. 브라우저 Notification API 또는 화면 내 알림 팝업. 2번의 소켓 이벤트를 받아서 알림을 트리거해요.
프로필 기능
내 프로필(이름, 상태메시지, 프로필 사진) 보기·수정. 프로필 사진 업로드는 FormData로 서버에 전송. 상태메시지는 Redux에 저장해서 전체에서 표시.

핵심 코드 예시
// 사용자 검색 - debounce로 불필요한 API 호출 방지
const [query, setQuery] = useState('');
const [results, setResults] = useState([]);

useEffect(() => {
  const timer = setTimeout(async () => {
    if (!query) return;
    const res = await api.get(`/users/search?q=${query}`);
    setResults(res.data);
  }, 300); // 300ms 대기
  return () => clearTimeout(timer);
}, [query]);

담당 파일 & 작성할 부분
server/app.js
Express 서버 진입점
포트 설정, CORS 미들웨어, 라우터(/api/auth, /api/messages) 연결. HTTP 서버 생성(지후 소켓 초기화를 위해 필요).
server/routes/auth.js
회원가입 · 로그인 API
POST /api/auth/register: 이름·이메일·비밀번호 받아서 DB 저장(bcrypt 암호화). POST /api/auth/login: 검증 후 JWT 토큰 발급해서 응답.
server/routes/messages.js
메시지 CRUD API
GET /api/messages/:roomId: 특정 방의 과거 메시지 목록 반환. POST /api/messages: 새 메시지 DB 저장. 영민의 React Query가 이 주소를 호출해요.
server/routes/users.js
사용자 검색 · 친구 API
GET /api/users/search?q=검색어: 이름·이메일로 검색 결과 반환. POST /api/friends/add: 친구 요청 저장. 서현이 이 주소를 프론트에서 호출해요.
server/models/User.js
사용자 DB 스키마
필드: name(String), email(String, unique), password(String, bcrypt 암호화), friends(ObjectId 배열), profileImage(String).
server/models/Message.js
메시지 DB 스키마
필드: content(String), sender(ObjectId → User 참조), room(ObjectId → Room 참조), isRead(Boolean), createdAt(Date).
server/models/Room.js
채팅방 DB 스키마
필드: participants(ObjectId 배열), lastMessage(String), updatedAt(Date). 서현의 RoomList가 이 데이터를 표시해요.
server/middleware/authMiddleware.js
JWT 인증 미들웨어
요청 헤더의 Authorization: Bearer 토큰을 검증. 유효하면 req.user에 유저 정보 저장 후 다음 단계로 넘김. 이게 없으면 비로그인으로도 메시지를 보낼 수 있어요.

2
지후 — 실시간 채팅 담당
Socket.IO로 서버-클라이언트 간 실시간 연결을 구축. 새로고침 없이 메시지가 오가게 해요.
담당 파일 & 작성할 부분
server/socket/index.js
소켓 이벤트 핸들러 (서버)
connection 이벤트 안에 joinRoom(방 입장), sendMessage(메시지 수신 후 방 전체 broadcast), disconnect(접속 해제), readMessage(읽음 처리) 이벤트를 작성해요.
server/app.js (기혁 협업)
소켓 초기화 코드 추가
기혁이 만든 app.js에 const httpServer = createServer(app) + const io = new Server(httpServer, { cors: {...} }) 두 줄을 추가해요. 기혁과 위치 협의 필요.
src/socket/socketClient.js
소켓 클라이언트 초기화 (프론트)
io('http://localhost:5000')로 소켓 인스턴스를 만들고 export해요. 이 파일을 import해서 어디서든 소켓을 사용할 수 있어요.
src/hooks/useSocket.js
소켓 React Hook (프론트)
컴포넌트가 마운트되면 소켓 연결, 언마운트되면 연결 해제하는 useEffect 로직. receiveMessage 이벤트를 듣고 Redux messageSlice에 새 메시지를 추가해요. 영민과 협업.
주로 쓰는 언어·기술
Socket.IO (서버) socket.io-client (프론트) JavaScript useEffect Hook
자주 쓰는 명령어
npm install socket.io최초 1회
server/ 폴더에서 실행. 서버 측 Socket.IO 설치. 기혁이 서버 기본 세팅 마친 뒤 실행해요.
npm install socket.io-client최초 1회
프론트 루트에서 실행. 클라이언트(브라우저) 측 Socket.IO 설치. 영민이 한꺼번에 설치할 때 같이 포함돼 있을 수도 있어요.
nodemon app.js매일 사용
server/ 폴더에서 실행. 소켓 코드 테스트 시 서버를 직접 켜서 확인해요. 기혁과 같이 실행해요.
npm run dev매일 사용
프론트 루트에서 실행. 프론트에서 소켓 연결이 잘 되는지 브라우저로 확인할 때 써요.
HTTP와 소켓의 차이를 기억해요. 과거 메시지 불러오기는 REST API(영민 담당), 새 메시지 실시간 전달은 Socket.IO(지후 담당). 두 개를 헷갈리지 말고 역할을 나눠요.

4
영민 — 프론트 기능 연동 담당
예지가 만든 UI와 기혁의 서버를 연결하는 역할. 이 담당이 없으면 버튼을 눌러도 아무 일도 안 일어나요.
담당 파일 & 작성할 부분
src/api/client.js
Axios 기본 설정 · 인터셉터
axios.create({ baseURL: import.meta.env.VITE_API_URL })로 인스턴스 생성. request 인터셉터에서 localStorage 토큰을 헤더에 자동 첨부. response 인터셉터에서 401 에러 시 자동 로그아웃.
src/api/authApi.js
로그인 · 회원가입 API 함수
login(email, password): POST 요청 후 받은 token을 localStorage에 저장하고 Redux authSlice에 유저 정보 저장. register(name, email, password): 회원가입 요청.
src/api/messageApi.js
메시지 API 함수
getMessages(roomId): 특정 방의 과거 메시지 목록 요청. sendMessage(content, roomId): 메시지 저장 요청. React Query의 useQuery·useMutation 안에서 사용해요.
src/api/userApi.js
사용자 · 친구 API 함수
searchUsers(query), addFriend(userId), getFriends() 함수 작성. 서현이 SearchUser·FriendList 컴포넌트에서 이 함수를 import해서 써요.
src/store/authSlice.js
로그인 상태 Redux Slice
state: user(유저 정보), token(JWT), isLoggedIn(boolean). actions: setUser, logout. 앱 시작 시 localStorage에서 토큰 불러와서 자동 로그인 체크.
src/store/messageSlice.js
메시지 상태 Redux Slice
state: messages(메시지 배열). actions: addMessage(소켓 새 메시지), setMessages(API로 불러온 목록). 지후의 소켓 훅에서 addMessage를 dispatch해요.
src/hooks/useMessages.js
메시지 불러오기 React Query 훅
useQuery({ queryKey: ['messages', roomId], queryFn: () => getMessages(roomId), refetchInterval: 3000 }). 로딩·에러 상태를 자동 관리하고 ChatWindow에서 사용해요.
src/App.jsx
Provider 설정 (가장 중요)
<QueryClientProvider>와 <Provider store={store}>로 앱 전체를 감싸는 코드 작성. 이게 없으면 React Query·Redux가 전혀 작동 안 해요.
.env (프론트)
프론트 환경 변수
VITE_API_URL=http://localhost:5000/api. Vite에서 환경변수는 반드시 VITE_로 시작해야 코드에서 import.meta.env.VITE_API_URL로 읽을 수 있어요.
주로 쓰는 언어·기술
Axios (API 요청) React Query Redux Toolkit async/await localStorage
자주 쓰는 명령어
npm install axios @tanstack/react-query react-redux @reduxjs/toolkit socket.io-client최초 1회
프론트 루트에서 실행. 영민·지후에게 필요한 핵심 패키지 전부를 한 번에 설치해요.
npm install처음 받을 때
Git에서 프로젝트 받은 직후 실행. package.json에 기록된 모든 라이브러리를 설치해요.
npm run dev매일 사용
개발 서버 시작. API 연결 테스트할 때도 꼭 켜두어요. 기혁의 백엔드 서버도 같이 켜야 API가 연결돼요.
토큰 만료 처리도 꼭 해요. Axios response 인터셉터에서 401 에러가 오면 localStorage.removeItem('token')하고 Redux logout dispatch 후 로그인 페이지로 리다이렉트. 이게 없으면 토큰 만료 후 무한 에러가 나요.
