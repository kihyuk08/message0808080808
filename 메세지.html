📖 코드 이해하기
🗺 이 앱의 전체 구조
이 메신저는 HTML + CSS + JavaScript 세 가지로 만들어졌습니다.

• HTML: 화면 구조 (버튼, 입력창, 메시지창…)
• CSS: 스타일 (색상, 크기, 배치…)
• JavaScript: 동작 (로그인, 메시지 전송, 실시간 통신…)
// JavaScript 코드 전체 흐름 1. login() → 이름 저장, 화면 전환 2. sendMessage() → 메시지 전송 + 저장 3. renderMessage() → 화면에 말풍선 추가 4. BroadcastChannel → 다른 탭에 전달 5. localStorage → 메시지 저장(영구)
💡 파이썬으로 비유하면: HTML = print(), CSS = 출력 형식, JavaScript = 함수·로직!
📡 BroadcastChannel — 탭 간 실시간 통신
여러 탭이 같은 "채널 이름"을 열면 서로 메시지를 주고받을 수 있어요. 실제 서버 없이도 탭끼리 통신할 수 있는 브라우저 내장 기능입니다.
// 채널 열기 (같은 이름 = 같은 채널) const ch = new BroadcastChannel('club-messenger'); // 다른 탭들에게 데이터 보내기 ch.postMessage({ type: 'chat', text: "안녕!", sender: "기혁" }); // 다른 탭에서 온 데이터 받기 ch.onmessage = (event) => { const data = event.data; console.log(data.sender, data.text); };
💡 파이썬의 socket과 비슷하지만, 같은 컴퓨터의 탭끼리만 통신해요. 진짜 서버가 있으면 Socket.IO를 쓰는데, 그건 가이드의 "지후" 역할입니다!
💾 localStorage — 브라우저에 데이터 저장
브라우저에 데이터를 저장하는 방법이에요. 새로고침하거나 브라우저를 껐다 켜도 데이터가 남아있어요. 메시지 기록이 여기에 저장됩니다.
// 저장 (문자열만 가능!) localStorage.setItem('key', '값'); // 객체를 저장하려면 JSON.stringify로 변환 const msgs = [{ text: "안녕" }]; localStorage.setItem('msgs', JSON.stringify(msgs) // 배열 → 문자열 ); // 불러오기 const loaded = JSON.parse( localStorage.getItem('msgs') ); // 문자열 → 배열
💡 파이썬의 json.dump() / json.load()와 같아요! JSON.stringify = 저장, JSON.parse = 불러오기.
🎨 DOM 조작 — 화면을 코드로 바꾸기
JavaScript로 HTML 요소를 찾고, 만들고, 추가합니다. 메시지 말풍선이 화면에 나타나는 것도 이 방법이에요.
// id로 요소 찾기 const area = document.getElementById('chat-messages'); // 새 div 요소 만들기 const bubble = document.createElement('div'); bubble.className = 'msg-bubble mine'; // CSS 클래스 적용 bubble.textContent = "안녕하세요!"; // 내용 // 화면(DOM)에 추가하기 area.appendChild(bubble);
💡 파이썬 tkinter의 Label() 만들고 pack()으로 붙이는 것과 같아요! createElement = 위젯 생성, appendChild = pack().
➜ 화살표 함수 & 템플릿 리터럴
JavaScript에서 함수를 짧게 쓰는 방법과, 문자열 안에 변수를 넣는 방법이에요.
// 일반 함수 function greet(name) { return "안녕, " + name + "!"; } // 화살표 함수 (같은 기능, 더 짧게) const greet = (name) => `안녕, ${name}!`; // ↑ 백틱(`) 사용 ↑ ${변수} // 파이썬 비교 # greet = lambda name: f"안녕, {name}!"
💡 JS의 `${변수}` = 파이썬의 f"문자열 {변수}". 백틱(`)은 키보드 왼쪽 위 숫자 1 왼쪽에 있어요!
👂 이벤트 리스너 — 사용자 행동 감지
버튼 클릭, 키보드 입력 등 사용자가 무언가를 하면 미리 등록한 함수가 실행됩니다.
// 클릭 이벤트 const btn = document.getElementById('send-btn'); btn.addEventListener('click', () => { sendMessage(); }); // 키보드 이벤트 input.addEventListener('keydown', (e) => { if (e.key === 'Enter' && !e.shiftKey) { e.preventDefault(); // 기본 동작 막기 sendMessage(); } });
💡 파이썬 tkinter의 button.bind('<Button-1>', handler)와 같아요! 어떤 이벤트가 생기면 어떤 함수를 실행할지 등록하는 거예요.
⏳ async/await — 기다리는 코드 (심화)
실제 서버에서 데이터를 받을 때처럼, "기다려야 하는" 작업에 사용해요. 가이드의 "영민" 역할에서 Axios + async/await를 씁니다.
// async: "이 함수엔 기다릴 것이 있어요" const loadMessages = async (roomId) => { // await: "응답이 올 때까지 여기서 기다려요" const res = await fetch(`/api/messages/${roomId}`); const data = await res.json(); return data; }; // 파이썬 requests와 비교 # res = requests.get(f"/api/messages/{room_id}") # data = res.json()
💡 파이썬의 requests.get()은 응답을 받을 때까지 그냥 기다리는데, JS의 await도 마찬가지예요. 단, await 없이 실행하면 응답도 안 왔는데 다음 줄이 실행되는 문제가 생겨요!
🚀 실제 서버로 확장하기
지금 이 앱은 같은 컴퓨터 탭끼리만 통신해요.
다른 컴퓨터와도 채팅하려면 가이드처럼 서버가 필요해요.
// 지금 (BroadcastChannel - 탭끼리만) channel.postMessage({ text: "안녕" }); // 실제 서버 (Socket.IO - 인터넷 어디서나) // 가이드의 "지후" 역할! const socket = io('http://서버주소:5000'); socket.emit('sendMessage', { text: "안녕" }); socket.on('receiveMessage', (data) => { renderMessage(data); });
💡 BroadcastChannel → Socket.IO로 바꾸면 진짜 메신저가 돼요. 코드 구조는 거의 같고 postMessage↔emit, onmessage↔on 만 바뀝니다!
